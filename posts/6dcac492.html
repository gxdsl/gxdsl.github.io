博客配置 Gitalk 评论系统

发表于2022-09-20|更新于2022-10-27
|阅读量:

Gitalk介绍

Gitalk是一个基于GitHub Issue和Preact开发的评论插件。Issue大家都很熟悉，Gitalk简单来说就是通过Issue来存储和管理评论的,使用 GitHub 登录支持Markdown。

Gitalk配置

创建 OAuth Apps 授权

点击此处注册OAuth Apps并填写相应内容：

项目说明
Application nameOAuth Apps应用的名字，可以根据自己喜好定义
Homepage URL个人博客仓库主页完整地址,https://github.com/XXX/XXX.github.io
Application descriptionOAuth Apps应用描述，可不填
Authorization callback URL回调地址，即登录完之后回到哪个地址

Homepage URL和Authorization callback URL的填写要严格按照https://的标准，如果是http://可能会出错。

创建完后找到Client ID和Client secrets，复制并记录，后续需要在_config.butterfly.yml中进行配置。
若忘记复制，可找到OAuth Apps应用进行复制，具体步骤如下：

  1. 点击右上角头像找到 Settings
  2. 在 Settings 页面中找到 Developer settings 并进入
  3. 进入OAuth Apps, 找到创建的应用

Butterfly 配置 Gitalk 评论系统

首先在博客文件夹的根目录下找到_config.butterfly.yml配置文件,在_config.butterfly.yml配置文件中找到comments属性， 在use中选择 Gitalk，其他配置根据个人需要选择,大致如下：


接着找到gitalk属性，将复制的Client ID和Client secret填入相应的位置，并输入对应的repo(仓库名称) 和admin(账号名称) ，其他参数根据个人需要设置即可
